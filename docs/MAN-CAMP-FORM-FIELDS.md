# Man Camp 2026 Fluent Forms Field Contract

This is the current public form and webhook contract for Man Camp 2026.

Fluent Forms + Square remain the payment layer. The backend still re-checks inventory and guardian/program rules on submission.

## Required public fields

Use these Fluent Forms `name` values where practical:

| Field | Required | Notes |
| --- | --- | --- |
| `first_name` | Yes | Registrant first name |
| `last_name` | Yes | Registrant last name |
| `email` | Yes | Confirmation and reconciliation |
| `phone` | Yes | Registrant phone |
| `age` | Yes | Used for minor and Young Men's validation |
| `age_group` | Yes | `adult` or `child` |
| `is_minor` | Recommended | Can be derived from age; keep explicit if form logic uses it |
| `is_guardian` | Recommended | Keep explicit when registrant is a guardian |
| `guardian_name` | Required for minors | Guardian must be at camp at all times |
| `guardian_phone` | Required for minors | Guardian contact |
| `guardian_email` | Required for minors | Guardian contact |
| `guardian_relationship` | Required for minors | Guardian relationship |
| `program_type` | Yes | `standard` or `young_mens` |
| `lodging_option_key` | Yes | Stable machine key for pricing/inventory |
| `lodging_option_label` | Yes | Human-readable option label preserved for audit/reconciliation |
| `attendance_type` | Recommended | `overnight` or `sabbath_only` |
| `shirt_size` | Yes | Inventory-tracked |
| `price_selected` | Yes | Amount shown to registrant and sent to Square |
| `payment_status` | Recommended | Preserve Square / Fluent Forms status if available |
| `payment_reference` | Recommended | Transaction or order reference |
| `payment_method` | Recommended | Defaults to Square via Fluent Forms |
| `frontend_total` | Recommended | Total generated by Fluent Forms before charge |
| `square_total` | Recommended | Charged total if Square metadata exposes it |
| `amount_paid` | Recommended | Persist explicit paid amount when available |
| `medical_notes` | Optional | Medical / special consideration placeholder |
| `special_considerations` | Optional | Additional notes |
| `notes` | Optional | General notes |

## Hidden fields used by the widget

| Field | Purpose |
| --- | --- |
| `people_json` | Primary attendee JSON payload |
| `roster_json` | Backward-compatible mirror of `people_json` |
| `attendee_count` | Count maintained by the widget |

## Program values

| Value | Rule |
| --- | --- |
| `standard` | Default registrant flow |
| `young_mens` | Ages 10-14 only, guardian info required |

## Payment mapping notes

Keep the public pricing fields explicit. Do not collapse them into backend-only derived values.

The plugin will preserve these from Fluent Forms / Square when present:

- `price_selected`
- `payment_status`
- `payment_reference`
- `payment_method`
- `frontend_total`
- `square_total`
- `amount_paid`
